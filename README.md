# 🎓 Student Management System

![Python](https://img.shields.io/badge/Python-3.x-blue)
![Flask](https://img.shields.io/badge/Flask-3.x-green)

A complete web-based **Student Management System** built with **Python Flask** and **MySQL**.
Features full CRUD operations, an interactive dashboard with charts, and covers all major SQL concepts including Views, Triggers, Stored Procedures, CTEs, Window Functions, and more.

## ✨ Features

### 📊 Dashboard
- Real-time statistics cards (Total Students, Subjects, Present, Absent)
- Subject-wise average marks bar chart
- Grade distribution pie chart
- Class-wise student horizontal bar chart
- Top 5 students leaderboard
- Quick action buttons

### 👨‍🎓 Student Management
- Add new students
- Edit student details
- Delete students with confirmation
- Live search and filter by name, email, class
- Server-side pagination (50 per page)

### 📝 Marks Management
- View all marks using INNER JOIN (students + marks + subjects)
- Color-coded marks display (Excellent / Good / Average / Fail)
- Progress bars for visual representation
- Automatic grade calculation via MySQL Function

### 📅 Attendance System
- Mark attendance via modal dialog
- Attendance summary using LEFT JOIN View
- Attendance percentage calculation
- Color-coded status indicators (Good / Average / Low)

### 📈 Reports (All SQL Concepts)
- **Performance Report** — using VIEW (`vw_student_performance`)
- **Student Ranking** — using Window Functions (RANK, DENSE_RANK, ROW_NUMBER)
- **Top Students** — using CTE (Common Table Expression)
- **Above Average** — using Nested Subquery

### 🛡️ Audit Log
- Auto-generated by MySQL Triggers
- Tracks all student additions
- Tracks all student deletions
- Color-coded action badges

## 🚀 Installation & Setup

### Step 1: Clone Repository
```bash
git clone https://github.com/yourusername/student-management-system.git
cd student-management-system
```

### Step 2: Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 3: Setup MySQL Database

**Option A — MySQL Workbench:**
1. Open MySQL Workbench
2. Open `database.sql`
3. Run the entire script (Ctrl + Shift + Enter)

**Option B — Command Line:**
```bash
mysql -u root -p < database.sql
```

### Step 4: Configure Database Connection
Open `connection.py` and update your MySQL password:
```python
def get_connection():
    conn = mysql.connector.connect(
        host     = "localhost",
        user     = "root",
        password = "your_mysql_password",   # ← Change this
        database = "student_db"
    )
    return conn
```

### Step 5: Test Database Connection
```bash
python connection.py
```
Expected output:
```
✅ Database Connected Successfully!
```

### Step 6: Run the Application
```bash
python app.py
```

### Step 7: Open in Browser
```
http://127.0.0.1:5000
```

## 🗃️ Database Schema

```
students
├── student_id    INT         PRIMARY KEY AUTO_INCREMENT
├── name          VARCHAR     NOT NULL
├── email         VARCHAR     UNIQUE NOT NULL
├── phone         VARCHAR
├── class         VARCHAR     NOT NULL
├── age           INT         CHECK (age BETWEEN 5 AND 30)
├── admission_date DATE       DEFAULT CURRENT_DATE
└── status        VARCHAR     DEFAULT 'Active'

subjects
├── subject_id    INT         PRIMARY KEY AUTO_INCREMENT
└── subject_name  VARCHAR     UNIQUE NOT NULL

marks
├── mark_id       INT         PRIMARY KEY AUTO_INCREMENT
├── student_id    INT         FOREIGN KEY → students(student_id)
├── subject_id    INT         FOREIGN KEY → subjects(subject_id)
├── marks         INT         CHECK (marks BETWEEN 0 AND 100)
└── exam_date     DATE        DEFAULT CURRENT_DATE

attendance
├── attend_id     INT         PRIMARY KEY AUTO_INCREMENT
├── student_id    INT         FOREIGN KEY → students(student_id)
├── date          DATE        NOT NULL
└── status        ENUM        ('Present', 'Absent')

audit_log
├── log_id        INT         PRIMARY KEY AUTO_INCREMENT
├── action        VARCHAR     NOT NULL
├── student_id    INT
├── log_time      DATETIME    DEFAULT NOW()
└── details       TEXT
```

## 🔒 Security Features

- ✅ **Parameterized queries** — prevents SQL Injection
- ✅ **ON DELETE CASCADE** — no orphan records
- ✅ **Transactions with Rollback** — safe data writes
- ✅ **CHECK constraints** — validates age (5–30) and marks (0–100)
- ✅ **UNIQUE constraint** on email — no duplicate students
- ✅ **try/except blocks** in all Python route functions
- ✅ **Form input validation** — required fields, type checks
- ✅ **Confirm dialog** before every delete action
- ✅ **Trigger-based audit** — all changes are logged automatically
